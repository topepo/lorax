# as.party.C5.0 validates tree parameter

    Code
      as.party(c5_model, tree = 0)
    Condition
      Error in `as.party()`:
      ! `tree` must be >= 1, not 0.

---

    Code
      as.party(c5_model, tree = 5)
    Condition
      Error in `as.party()`:
      ! `tree` must be between 1 and 1, not 5.

---

    Code
      as.party(c5_model, tree = c(1, 2))
    Condition
      Error in `as.party()`:
      ! `tree` must be a single integer, not a double vector.

---

    Code
      as.party(c5_model, tree = "1")
    Condition
      Error in `as.party()`:
      ! `tree` must be a single integer, not a string.

# as.party.C5.0 works with boosted models

    Code
      as.party(c5_boost, tree = 0, data = penguins)
    Condition
      Error in `as.party()`:
      ! `tree` must be >= 1, not 0.

---

    Code
      as.party(c5_boost, tree = 11, data = penguins)
    Condition
      Error in `as.party()`:
      ! `tree` must be between 1 and 10, not 11.

# as.party.C5.0 rejects rule-based models

    Code
      as.party(c5_rules, tree = 1)
    Condition
      Error in `as.party()`:
      ! C5.0 rule-based models are not supported. Use tree-based models only.
